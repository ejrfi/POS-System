Baik, saya mengerti. Anda ingin **fitur upload file gambar fisik** (bukan link internet) untuk produk.

Ini memerlukan perubahan signifikan karena kita perlu menangani file upload di sisi server (backend) dan frontend.

### Rencana Implementasi Upload Gambar Fisik

#### 1. Backend (Server)
*   **API Endpoint Baru**: Buat endpoint `POST /api/upload` untuk menerima file gambar.
*   **Middleware**: Gunakan library `multer` (atau sejenisnya untuk Node.js) untuk memproses `multipart/form-data`.
*   **Penyimpanan**: Simpan file yang diupload ke folder publik di server (misal: `uploads/`).
*   **Response**: Endpoint akan mengembalikan URL/path relatif gambar yang berhasil diupload (contoh: `/uploads/gambar-kopi.jpg`).

#### 2. Frontend (Inventory)
*   **Update Form Produk**: Ganti input teks "Image URL" dengan input tipe file (`<input type="file" />`).
*   **Logika Upload**:
    *   Saat user memilih file, frontend akan mengirim file tersebut ke endpoint `/api/upload`.
    *   Setelah sukses, server mengembalikan URL gambar.
    *   URL tersebut kemudian disimpan ke field `image` pada data produk (saat tombol "Simpan" diklik).
*   **Preview**: Tampilkan preview gambar yang dipilih sebelum disimpan.

#### 3. Tampilan (Inventory & POS)
*   **Inventory Table**: Tambahkan kolom kecil (thumbnail) untuk menampilkan gambar produk yang tersimpan.
*   **POS Card**: Pastikan komponen `ProductCard` bisa merender gambar dari path lokal server (misal: `http://localhost:5000/uploads/...`).
*   **POS List View**: Implementasikan tampilan daftar (list) sesuai permintaan sebelumnya, dengan thumbnail gambar kecil di sebelah kiri.

### Langkah Kerja
1.  **Setup Server**: Install `multer`, buat folder `uploads`, dan konfigurasi endpoint `/api/upload`.
2.  **Update Inventory Page**: Ganti input URL dengan komponen File Upload + Preview.
3.  **Update Product Display**: Pastikan gambar muncul di tabel Inventory dan POS (baik Grid maupun List view).
4.  **Implementasi List View POS**: Tambahkan toggle Grid/List di halaman POS.
