**Analisis Mendalam & Akar Masalah (Terkonfirmasi):**
Server berjalan di **port 5001**, tetapi aplikasi frontend (Vite) secara default berjalan di **port 5173**.

1. **Request API Gambar**: Saat aplikasi dibuka di `http://localhost:5173` (port default Vite), request ke `/api/uploads/gambar.jpg` akan diarahkan ke `http://localhost:5173/api/uploads/gambar.jpg`.
2. **Kegagalan**: Karena tidak ada proxy yang mengarahkan request `/api/uploads` dari port 5173 ke port 5001, request ini gagal (404 Not Found di server Vite).
3. **Solusi Sebelumnya Gagal**: Penghapusan proxy di `vite.config.ts` sebelumnya justru memperburuk keadaan karena memutus jembatan antara frontend (5173) dan backend (5001).

**Solusi "100% Ampuh" (Fix Proxy & Base URL):**
Kita harus mengembalikan konfigurasi proxy di `vite.config.ts` agar frontend (port 5173) bisa "berbicara" dengan backend (port 5001), KHUSUSNYA untuk request gambar dan API.

**Rencana Perbaikan:**

1. **Restore Proxy (`vite.config.ts`)**:

   * Tambahkan kembali proxy untuk `/api` dan `/api/uploads` (path baru gambar).

   * Set target proxy ke `http://localhost:5001` (sesuai konfirmasi Anda).

   * Pastikan `changeOrigin: true` agar host header sesuai.

2. **Verifikasi URL Gambar**:

   * Pastikan kode frontend tidak melakukan hardcode `http://localhost:5000` atau port lain. Cukup gunakan relative path `/api/uploads/...` agar proxy yang menangani sisanya.
