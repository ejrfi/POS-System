**Analisis Mendalam & Kesimpulan Akhir:**
Setelah memperbaiki backend dan proxy, jika gambar masih tidak muncul di frontend, masalahnya 99% terletak pada **Frontend yang tidak tahu cara mengakses file di server lain (beda port)**.

**Fakta:**

1. Frontend berjalan di `http://localhost:5173`.
2. Backend berjalan di `http://localhost:5001`.
3. URL gambar di database: `/api/uploads/gambar.jpg`.
4. Saat frontend memuat `<img src="/api/uploads/gambar.jpg" />`, browser mencoba mengambil dari `http://localhost:5173/api/uploads/gambar.jpg`.
5. Meskipun proxy sudah diset, terkadang browser atau environment tertentu tidak meneruskan request gambar dengan benar lewat proxy dev server.

**Solusi "100% Ampuh" (Absolute URL di Frontend):**
Kita tidak bisa mengandalkan relative path `/api/uploads/...` jika proxy bermasalah. Kita harus **memaksa** frontend untuk mengambil gambar langsung dari sumbernya: Backend (Port 5001).

**Rencana Implementasi:**

1. **Buat Helper URL**: Di file `client/src/lib/utils.ts`, buat fungsi `getImageUrl(path: string)` yang akan mendeteksi apakah path adalah relative. Jika ya, tambahkan prefix `http://localhost:5001`.

   * *Catatan:* Ini hardcode untuk dev environment. Untuk produksi nanti bisa disesuaikan, tapi ini solusi instan untuk masalah Anda sekarang.
2. **Terapkan di UI**: Update `ProductCard.tsx` dan `Inventory.tsx` untuk menggunakan helper ini saat merender `src`.
