**Analisis Mendalam & Temuan Kritis:**

1.  **Misteri 404 vs HTML**: Jika backend tidak menemukan gambar di `/api/uploads/xxx.jpg`, request tersebut akan "tembus" ke middleware Vite/Frontend. Akibatnya, alih-alih mengembalikan error 404, server malah mengembalikan file `index.html` (halaman utama React). Browser mencoba merender HTML ini sebagai gambar, gagal, dan akhirnya memicu `onError`.
2.  **Path Upload**: Kita sudah menggunakan absolute path. TAPI, ada kemungkinan path tempat script dijalankan (`process.cwd()`) berbeda saat dijalankan oleh `npm run dev` (via `cross-env`).

**Solusi "100% Ampuh" (Diagnostic & Fix):**

Kita akan melakukan 2 hal sekaligus: Memastikan path benar & Mencegah request gambar "bocor" ke frontend.

1.  **Debugging Path Absolut (Lagi)**:
    Saya akan membuat script kecil `check-paths.js` yang akan dijalankan *tepat sebelum* server start. Script ini akan mencetak `process.cwd()` dan isi folder `uploads` secara gamblang ke terminal. Ini akan mengonfirmasi apakah file gambar *benar-benar ada* di tempat yang server cari.

2.  **Middleware 404 Khusus API**:
    Di `server/routes.ts`, tepat setelah baris `app.use('/api/uploads', ...)`, saya akan menambahkan middleware "penjaga gawang":
    ```typescript
    app.use('/api/uploads', (req, res) => {
      // Jika request sampai sini, berarti express.static tidak menemukan filenya
      res.status(404).json({ message: "Image not found on server" });
    });
    ```
    Ini akan mencegah request gambar yang hilang agar tidak jatuh ke Vite/Frontend, sehingga kita bisa melihat error 404 yang *sebenarnya* di Network Tab browser (bukan 200 OK dengan isi HTML).

3.  **Verifikasi Nama File**:
    Saya curiga ada perbedaan nama file di database vs di disk (misal: spasi, ekstensi case-sensitive `.JPG` vs `.jpg`). Script checker akan mengungkap ini.
