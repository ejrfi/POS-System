**Analisis Masalah:**
Meskipun data gambar sudah masuk ke database dan logika kode tampaknya benar, gambar tetap tidak muncul di Inventory dan POS. Masalah ini kemungkinan besar disebabkan oleh **Path Gambar yang Tidak Konsisten** antara apa yang disimpan di database dengan bagaimana browser mencoba mengaksesnya.

**Detail Teknis:**
1.  **Path di Database**: Upload gambar menghasilkan path relatif seperti `/uploads/filename.jpg`.
2.  **Akses Browser**:
    *   Di mode development (Vite), browser mengakses `http://localhost:5173/uploads/filename.jpg`.
    *   Karena kita sudah setup proxy di `vite.config.ts`, request ini *seharusnya* diteruskan ke backend (`http://localhost:5000/uploads/...`).
3.  **Kemungkinan Penyebab Kegagalan**:
    *   **Browser Caching**: Browser mungkin menyimpan respons 404 lama.
    *   **Path Absolute vs Relative**: Jika di database tersimpan path Windows (misal `uploads\file.jpg` dengan backslash), browser tidak bisa membacanya dengan benar sebagai URL.
    *   **Missing Base URL**: Kadang diperlukan prefix URL lengkap (misal `http://localhost:5000/uploads/...`) jika proxy tidak berjalan mulus.

**Rencana Perbaikan:**

1.  **Debugging Path (Inventory & POS)**:
    *   Saya akan memodifikasi `Inventory.tsx` dan `ProductCard.tsx` untuk melakukan logging path gambar yang diterima dari database ke console browser. Ini akan memastikan apakah path yang diterima benar (misal `/uploads/gambar.jpg`).

2.  **Perbaikan Rendering Gambar (Fallback & Path Handling)**:
    *   Mengupdate komponen `ProductCard` dan tabel `Inventory` untuk:
        *   Menangani path gambar dengan lebih robust (memastikan diawali `/`).
        *   Menambahkan **error handling** pada tag `<img>` (event `onError`) untuk menampilkan gambar placeholder jika gambar gagal dimuat.
        *   Menambahkan placeholder default di `ProductCard` agar tampilan kartu tetap konsisten tingginya meskipun tidak ada gambar.

3.  **Verifikasi Ulang Proxy**:
    *   Memastikan konfigurasi proxy di `vite.config.ts` sudah aktif dan benar (sudah dilakukan di langkah sebelumnya, tapi perlu restart server dev jika belum).
