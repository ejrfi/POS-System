**Analisis Mendalam & Kesimpulan Akhir (Kenapa Error & Gambar Tidak Muncul):**

1.  **Error `PayloadTooLargeError`**:
    Server Express secara default membatasi ukuran request body (JSON) hanya **100KB**.
    Karena kita sekarang mengirim gambar sebagai Base64 string (yang ukurannya bisa 1-2MB) langsung di dalam payload JSON saat membuat/mengupdate produk, request tersebut ditolak oleh server sebelum masuk ke logika database.

2.  **Gambar Tidak Muncul (Format URL)**:
    Helper `getImageUrl` di frontend saat ini diprogram untuk menambahkan slash `/` di depan string yang tidak diawali `http`.
    *   Input: `data:image/jpeg;base64,...`
    *   Output Salah: `/data:image/jpeg;base64,...` (Browser menganggap ini relative path, bukan data URI).
    *   Akibatnya: Gambar rusak.

**Solusi "100% Ampuh" (Fix Limit & Helper):**

1.  **Tingkatkan Limit Payload Server**:
    Ubah konfigurasi `express.json()` dan `express.urlencoded()` di `server/index.ts` untuk menerima payload besar (misal **50MB**). Ini wajib agar Base64 bisa masuk.

2.  **Perbaiki Helper Frontend**:
    Update `getImageUrl` di `client/src/lib/utils.ts` untuk mendeteksi prefix `data:` dan mengembalikannya apa adanya tanpa tambahan slash.

3.  **Verifikasi Database**:
    Pastikan kolom `image` di database (MySQL) benar-benar mampu menampung string panjang. Sebelumnya kita sudah ubah schema, tapi kita perlu memastikan tipe datanya `LONGTEXT` di database fisik. Saya akan buat script migrasi kecil untuk memaksanya.

**Rencana Eksekusi:**
1.  **Server**: Naikkan limit body parser ke 50MB.
2.  **Frontend**: Update `getImageUrl` support Base64.
3.  **Database**: Jalankan query `ALTER TABLE products MODIFY image LONGTEXT` untuk jaminan 100% kapasitas penyimpanan.
