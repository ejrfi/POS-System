**Analisis Masalah:**
Gambar produk tidak muncul di frontend (POS/Inventory) meskipun upload berhasil.

**Penyebab:**
Dalam arsitektur pengembangan saat ini (Vite + Express), request untuk gambar statis (`/uploads/...`) sering kali tidak ditangani dengan benar oleh server dev Vite jika tidak ada konfigurasi proxy eksplisit. Vite mencoba melayani file tersebut sebagai aset statis frontend, padahal file tersebut berada di backend (server Express).

**Rencana Perbaikan:**

1.  **Konfigurasi Proxy Vite (`vite.config.ts`)**:
    *   Menambahkan aturan proxy untuk path `/uploads`. Ini akan memberitahu Vite untuk meneruskan setiap request yang diawali dengan `/uploads` ke server backend (biasanya berjalan di port 5000), sama seperti request `/api`.

2.  **Verifikasi Path Gambar**:
    *   Memastikan bahwa URL gambar yang disimpan di database diawali dengan `/uploads/` (relatif terhadap root server), bukan path absolut file sistem. (Sudah diverifikasi di langkah sebelumnya: `server/routes.ts` menyimpan sebagai `/uploads/filename`).

3.  **Verifikasi Middleware Statis Server**:
    *   Memastikan baris `app.use('/uploads', express.static('uploads'));` di `server/routes.ts` sudah benar dan ditempatkan sebelum route handler lainnya. (Sudah diverifikasi: ada di baris 103).

**Langkah Implementasi:**
Hanya satu langkah utama yang diperlukan: Mengupdate `vite.config.ts` untuk menambahkan proxy `/uploads`.
