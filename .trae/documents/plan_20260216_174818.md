**Analisis Mendalam & Koreksi Arah (Konteks Electron):**
User menyatakan: "tujuan saya nanti aplikasi ini menjadi aplikasi desktop, saya akan menggunakan Electron".

Ini menjelaskan kenapa user sangat peduli dengan akses port `localhost:5001` secara langsung (karena Electron biasanya meload `index.html` dari server lokal atau file lokal, dan backend harus bisa diakses dari konteks tersebut).

**Akar Masalah Saat Ini:**
1.  **Helper URL Salah**: Kode `getImageUrl` saya sebelumnya yang melakukan hardcode `http://localhost:5001` **adalah ide buruk** untuk Electron, karena di Electron, origin frontend mungkin bukan `localhost` (bisa `file://` atau custom protocol).
2.  **Relative Path Paling Aman**: Untuk aplikasi desktop/Electron, cara paling aman dan portabel adalah menggunakan **Relative Path** yang benar, atau jika terpaksa absolute, harus dinamis.
3.  **Vite Proxy Tidak Relevan**: Di Electron production, tidak ada Vite Dev Server. Backend (Express) harus melayani API dan Static Files (Gambar) secara mandiri.

**Solusi Final (Electron-Ready):**
Kita harus membuat aplikasi ini agnostik terhadap di mana ia dijalankan (browser dev, browser prod, atau Electron).

**Rencana Implementasi:**

1.  **Hapus Hardcoded URL di Frontend (`client/src/lib/utils.ts`)**:
    Kembalikan `getImageUrl` menjadi fungsi sederhana yang hanya memastikan path diawali dengan `/`.
    *   *Kenapa?* Karena saat Anda membuka `http://localhost:5001` (seperti yang Anda lakukan sekarang), browser otomatis menganggap `/api/uploads/gambar.jpg` sebagai `http://localhost:5001/api/uploads/gambar.jpg`. Ini **pasti berhasil**.

2.  **Verifikasi Backend Serving (`server/routes.ts`)**:
    Pastikan `app.use('/api/uploads', express.static(...))` tetap ada dan dikonfigurasi dengan benar (sudah OK dari langkah sebelumnya).

3.  **Pastikan Folder Upload Ada**:
    Saya akan menambahkan script kecil (atau instruksi manual) untuk memastikan folder `uploads` dan isinya benar-benar ada di lokasi yang diharapkan server.

**Kenapa ini solusi terbaik?**
*   **Dev Mode (Vite 5173)**: Proxy di `vite.config.ts` (yang sudah kita perbaiki ke port 5001) akan menangani request `/api/uploads/...`.
*   **Preview Mode (Express 5001)**: Request `/api/uploads/...` langsung ditangani Express.
*   **Electron**: Request ke backend (biasanya `localhost` server yang dispawn oleh Electron) juga akan berhasil.
